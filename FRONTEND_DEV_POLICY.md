# FRONTEND_DEV_POLICY

## Роли
- **Lovable** отвечает за дизайн, верстку и UX.
- **Lovable** работает только в ветке **`lovable-dev`** (через PR в `lovable-dev`).
- **Codex** отвечает за инженерные задачи frontend: API-интеграции, типы, обработку ошибок, тесты и документацию.

## Ограничения для Codex
- Codex **не меняет** layout/styles/components без явного запроса.
- Любые изменения UI (включая «мелкие визуальные») выполняются только по отдельной явной задаче.

## Контракт (backend-first)
- Любые изменения API-контракта вносятся через canonical-контракт в backend.
- Frontend не является источником истины по контракту.
- Сначала merge изменений контракта в backend, затем синхронизация frontend.

## Ветки
- Рабочий поток: `feature/*` → Pull Request → `lovable-dev`.
- Прямые push в `main` запрещены.
