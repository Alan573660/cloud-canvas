# FRONTEND_DEV_POLICY

## Роли и зона ответственности
- **Lovable**: дизайн, верстка, UX-потоки, визуальные правки интерфейса.
- **Lovable** работает только в ветке **`lovable-dev`** (и PR в нее).
- **Codex**: инженерные задачи frontend — API-интеграции, типы, обработка ошибок, тесты, документация.

## Ограничения для Codex
- Codex **не меняет** layout/styles/components без явного запроса.
- Любые UI-изменения (включая «небольшие визуальные») — только по отдельному подтвержденному запросу.

## Контракт и backend-first правило
- Любые изменения API-контракта считаются **backend-first**.
- Canonical источник контракта находится в backend-репозитории и правится только там.
- Frontend адаптируется к контракту только после принятого backend PR.

## Ветки и процесс
- Рабочий поток: `feature/*` → Pull Request → `lovable-dev`.
- Прямые push в `main` запрещены.
