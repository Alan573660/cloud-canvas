# PR_POLICY
Canonical PR policy: maxim-saas/PR_POLICY.md

- Все изменения делаются через feature-ветки: `feature/*`.
- Поток: `feature/*` → Pull Request → `lovable-dev`.
- Прямые push в `main` запрещены.
- PR должен быть атомарным и с понятной областью изменений.
- Шаблон описания PR обязателен:
  - Что изменилось
  - Как проверить
  - Риски
  - Handoff
- Если меняется контракт/поведение API, блок **Handoff** обязателен.
- Если меняются роли/договоренности по процессу, обновить соответствующие docs в этом репозитории.
- Не смешивать UI-редизайн и инженерные backend-integration правки в одном PR.
