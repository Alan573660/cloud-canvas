# PR_POLICY
Canonical PR policy: maxim-saas/PR_POLICY.md

- Используем только feature-ветки: `feature/*`.
- Маршрут изменений: `feature/*` → PR → `lovable-dev`.
- PR target по умолчанию: `lovable-dev`.
- Прямые push в `main` запрещены.
- PR должен быть атомарным и сфокусированным.
- Шаблон PR обязателен: **Что изменилось / Как проверить / Риски / Handoff**.
- Если меняется контракт или поведение, блок **Handoff** обязателен.
- Не смешивать UI-редизайн и инженерные интеграционные правки в одном PR.
- При изменении процесса обновлять соответствующие docs в репозитории.
