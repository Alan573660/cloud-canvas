# Нормализация прайса — Продуктовое ТЗ (UX-first)

> Версия: 1.0  
> Дата: 2026-02-09  
> Фокус: Пользовательские сценарии, юзабилити, структура интерфейса  
> Не содержит: технических решений, выбора стека, схем БД

---

## 1. Общая концепция

### 1.1. Проблема

Поставщики кровельных материалов присылают прайс-листы в произвольных форматах:
- Разные названия одного и того же товара ("С-20 0.5 PE RAL3005", "Профнастил С20 полиэстер зелёный мох")
- Разная структура файлов (колонки, вложенные заголовки, объединённые ячейки)
- Отсутствие стандартизированных атрибутов (толщина в заголовке листа, а не в строке)
- Цвета указаны словами ("шоколад"), кодами (RAL8017) или вообще не указаны

**Цель системы**: превратить любой входящий прайс в структурированный каталог с единой схемой атрибутов за минимальное количество действий пользователя.

### 1.2. Принцип работы

Система работает по принципу **«ИИ делает — человек подтверждает»**:

1. ИИ автоматически распознаёт и заполняет ~90% атрибутов
2. Для оставшихся ~10% система задаёт точечные вопросы
3. Пользователь подтверждает паттернами, а не строка за строкой
4. Один подтверждённый паттерн может исправить сотни строк

### 1.3. Ключевой принцип UX

> **Пользователь никогда не редактирует отдельные строки.**  
> Он работает с группами (кластерами) похожих товаров и применяет решения сразу ко всей группе.

## 2. Путь пользователя (End-to-End)

```
┌─────────────┐    ┌──────────────┐    ┌───────────────┐    ┌──────────────┐    ┌──────────────┐
│  1. ЗАГРУЗКА │───▶│ 2. МАППИНГ   │───▶│ 3. НОРМАЛИЗА- │───▶│ 4. ПРОВЕРКА  │───▶│ 5. ПУБЛИКА-  │
│  прайса      │    │  колонок     │    │    ЦИЯ        │    │  и правки    │    │    ЦИЯ       │
└─────────────┘    └──────────────┘    └───────────────┘    └──────────────┘    └──────────────┘
     5 сек             30 сек             автомат.              2–10 мин            1 клик
```

### Шаг 1. Загрузка прайса
- Пользователь загружает файл (CSV, XLSX, PDF)
- Система показывает превью первых строк
- Автоматически определяет формат и кодировку

### Шаг 2. Маппинг колонок
- Система предлагает автоматический маппинг (какая колонка = какое поле)
- Пользователь корректирует при необходимости
- Для повторных загрузок от того же поставщика — маппинг запоминается

### Шаг 3. Нормализация (автоматическая)
- ИИ классифицирует каждый товар по категории
- Извлекает атрибуты из названий
- Сопоставляет цвета с каталогом RAL
- Определяет покрытие и толщину

### Шаг 4. Проверка и правки (основной экран — см. раздел 3)
- Пользователь видит результат нормализации
- Работает с кластерами через визуальный интерфейс или AI-чат
- Подтверждает или корректирует предложения ИИ

### Шаг 5. Публикация
- Один клик «Обновить каталог»
- Данные становятся доступны для расчётов и продаж

### 3.2. Левая панель — Навигация (Дерево кластеров)

**Назначение**: позволяет быстро найти нужную группу товаров.

**Иерархия дерева**:
```
📦 Все товары (1204)
├── 🏗 Профнастил (687)
│   ├── С-8 (45)      ✅
│   ├── С-20 (120)     🔴 3 вопроса
│   │   ├── 0.4 мм (30)
│   │   ├── 0.45 мм (40)   🔴
│   │   └── 0.5 мм (50)
│   ├── С-21 (98)      ✅
│   ├── МП-20 (67)     ✅
│   ├── НС-35 (84)     🔴 1 вопрос
│   ├── Н-60 (43)      ✅
│   └── Н-75 (30)      ✅
├── 🏠 Металлочерепица (312)
│   ├── Монтеррей (156)
│   ├── Супермонтеррей (89)
│   └── Каскад (67)
├── 📐 Доборные элементы (158)
│   ├── Планки (72)
│   ├── Коньки (34)
│   ├── Отливы (28)
│   └── Прочее (24)
└── 🧱 Сэндвич-панели (47)
    ├── Стеновые (28)
    └── Кровельные (19)
```

**Поведение**:
- Клик по узлу = фильтрация таблицы справа
- Счётчики показывают количество товаров в группе
- Индикаторы ✅/🔴 — статус нормализации (все поля заполнены / есть пробелы)
- Клик по 🔴 — переход к первому нерешённому вопросу

**Уровни вложенности для профнастила и металлочерепицы**:
```
Категория → Профиль → Толщина → Покрытие → Цвет/RAL
```

**Для доборных элементов и сэндвич-панелей** — упрощённая группировка:
```
Категория → Тип изделия → Покрытие
```

### 3.3. Центральная область — Таблица товаров

**Назначение**: показать текущее состояние нормализации для выбранного кластера.

**Колонки таблицы**:

| Колонка | Описание | Редактируемость |
|---------|----------|-----------------|
| Исходное название | Оригинальный текст из прайса | Только чтение |
| Профиль | С-20, Монтеррей и т.д. | Автоматически |
| Толщина, мм | 0.4 / 0.45 / 0.5 / 0.7 | Групповое изменение |
| Покрытие | Полиэстер, Пластизол... | Групповое изменение |
| Цвет / RAL | RAL3005, Zn | Групповое изменение |
| Раб. ширина, мм | 1100 | Из базы профилей (авто) |
| Полн. ширина, мм | 1150 | Из базы профилей (авто) |
| Цена, ₽/м² | 450.00 | Из прайса (только чтение) |
| Статус | ✅ / 🔴 | Автоматический |

**Подсветка изменений (До → После)**:
- Ячейки, которые ИИ заполнил автоматически, подсвечиваются зелёным фоном
- Пустые обязательные ячейки подсвечиваются красной рамкой
- При наведении на заполненную ячейку — тултип: «Определено автоматически из "С-20 0.5 PE зел.мох"»

**Фильтры над таблицей**:
- По статусу: Все / ✅ Готовые / 🔴 Требуют внимания
- Поиск по названию

**Массовые действия**:
- Выделить несколько строк → кнопка «Задать покрытие для выбранных»
- Выделить всё в кластере → «Применить ко всей группе»

## 5. AI-чат ассистент — подробное описание

### 5.1. Концепция

AI-чат — это **командный инструмент**, а не свободный диалог. Пользователь отдаёт команды на естественном языке, ИИ генерирует конкретные действия (патчи) с превью, которые пользователь подтверждает или отклоняет.

### 5.2. Расположение

Чат расположен **внизу основной области** (под таблицей) и может быть:
- Свёрнут (видна только полоска с иконкой 💬 и количеством нерешённых вопросов)
- Развёрнут (занимает ~30% высоты экрана)

Чат **контекстно-зависим**: он знает, какой кластер сейчас выбран в дереве слева.

```
   → Применить толщину 0.5 мм для 84 товаров профиля НС-35?
```

### 5.4. Типы команд, которые понимает AI-чат

| Тип команды | Пример | Действие |
|-------------|--------|----------|
| Назначить покрытие | «MatPE — это матовый полиэстер» | Массовое изменение поля «покрытие» |
| Назначить цвет | «зел.мох — это RAL6005» | Массовое изменение поля «цвет/RAL» |
| Установить ширину | «С-20: рабочая 1100, полная 1150» | Установка ширин для профиля |
| Установить толщину | «Все С-20 без толщины — это 0.45» | Заполнение пропущенных толщин |
| Запрос статистики | «Сколько товаров без цвета?» | Информационный ответ |
| Поиск | «Покажи все товары с "вишня"» | Фильтрация таблицы |
| Исправление | «Нет, это не RAL6002, а RAL6005» | Откат и переназначение |
| Подтверждение | «Да», «Применить», «Ок» | Подтверждение предложенного патча |
| Отмена | «Нет», «Отменить», «Не надо» | Отмена предложенного патча |

### 5.5. Правила поведения AI

1. **Всегда показывать превью** — ни одно изменение без предварительного показа «что будет изменено»
2. **Показывать количество** — «затронет 47 товаров» (не «несколько» или «много»)
3. **Показывать примеры** — минимум 3 строки «До → После»
4. **Одно действие за раз** — не предлагать исправить покрытие и цвет одновременно
5. **Запоминать решения** — если пользователь сказал «MatPE = матовый полиэстер», при следующем импорте этого поставщика — применять автоматически
6. **Не угадывать** — если уверенность ниже 80%, лучше спросить, чем назначить
7. **Говорить на языке пользователя** — не «RAL Classic palette sRGB value», а «RAL6005 — зелёный мох, популярный цвет для кровли»

## 8. Масштабирование на другие отрасли

### 8.1. Принцип

Текущая система создана для кровельных материалов, но архитектура позволяет добавлять шаблоны для других отраслей:

| Отрасль | Категории | Ключевые атрибуты |
|---------|-----------|-------------------|
| **Кровля** (текущая) | Профнастил, Металлочерепица, Доборные, Сэндвич | Профиль, Толщина, Покрытие, RAL, Ширины |
| **Метизы** | Болты, Гайки, Шайбы, Саморезы | Диаметр, Длина, Шаг резьбы, Покрытие, Класс прочности |
| **Трубы** | Профильные, Круглые, ВГП | Диаметр, Стенка, Длина, Марка стали, ГОСТ |
| **Арматура** | Стержневая, Сетка, Проволока | Диаметр, Класс (А400, А500), Длина, Марка стали |
| **Фасады** | Сайдинг, Фасадные панели, Подсистемы | Тип, Размер, Покрытие, Цвет |

### 8.2. Для каждой отрасли определяется

1. **Каталог категорий** — иерархия (аналог дерева кластеров)
2. **Обязательные атрибуты** — что нужно заполнить для каждой категории
3. **Справочник синонимов** — как одно и то же называется в разных прайсах
4. **Справочник стандартов** — ГОСТ, ТУ, DIN и т.д.
5. **Правила валидации** — допустимые значения (толщина от 0.3 до 5.0 мм)

## 10. Требования к качеству

### 10.1. Метрики

| Метрика | Целевое значение |
|---------|-----------------|
| Автоматическое определение категории | ≥ 95% |
| Автоматическое определение профиля | ≥ 98% |
| Автоматическое определение покрытия | ≥ 85% |
| Автоматическое определение RAL | ≥ 90% |
| Количество кликов на 1000 товаров (повторный прайс) | ≤ 5 |
| Время нормализации прайса 1000 строк (новый поставщик) | ≤ 10 минут |
| Время нормализации прайса 1000 строк (повторный) | ≤ 2 минуты |

### 10.2. Пользовательский опыт

- Пользователь **не должен** вводить данные вручную, если система может их определить
- Пользователь **не должен** работать со строками по одной — только с группами
- Каждое действие пользователя должно влиять на **максимальное количество строк**
- Интерфейс должен работать **плавно** на прайсах до 10 000 строк
- Голосовой ввод должен работать для основных команд на русском языке
